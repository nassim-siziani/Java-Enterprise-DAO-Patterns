## 🎯 Project Overview

This project demonstrates the implementation of a robust, decoupled enterprise persistence layer in Java. It showcases how to isolate business logic from database infrastructure using advanced architectural design patterns. 

The application manages an organizational structure (Employees and Teams) and allows seamless switching between three entirely different persistence mechanisms (Binary Serialization, Raw SQL via JDBC, and ORM via JPA/Hibernate) without altering the core domain logic.

### **✨ Architectural Highlights & Patterns Used:**
* **Data Access Object (DAO):** Standardized generic interfaces (`DAO<T>`) abstracting CRUD operations, utilizing `Optional` to handle null-safety natively.
* **Abstract Factory Pattern:** Dynamic switching between persistence environments (`DAOFactorySerialize`, `DAOFactoryJDBC`, `DAOFactoryJPA`) using a central producer.
* **Builder Pattern:** Clean, immutable instantiation of complex `Employee` entities containing dynamic maps and collections.
* **Composite Pattern:** Hierarchical management of `Team` and `OrganizationElement` objects with recursive infinite-loop protection and custom iterators.

## 📁 Persistence Implementations

### 1. Java Serialization Layer
Stores objects as raw binary `.ser` files using standard `ObjectOutputStream`. Acts as the baseline persistent storage solution.

### 2. JDBC Layer (Raw SQL)
Communicates directly with the Apache Derby embedded database using raw SQL queries (`PreparedStatement`, `ResultSet`). Demonstrates low-level database connection management and manual object mapping.

### 3. JPA / Hibernate Layer (ORM)
The most advanced layer. Utilizes Java annotations (`@Entity`, `@Table`, `@ElementCollection`, `@MapKeyEnumerated`) to automatically map complex object graphs (including inner Lists and HashMaps) to relational database tables, handling cascading saves and retrievals transparently.
